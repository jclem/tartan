tartan

tartan is a simple, flexible grid system packaged as a Ruby gem.

The primary goal of tartan is to create a grid that can be adapted to a diverse set of layout scenarios.

Where wrapping content in parent row containers is an option, tartan supports positioning content via floats. Where this is not possible, tartan supports positioning content via inline-block, eliminating the need to wrap content in row containers. This is especially useful in fluid/responsive layouts.

Install

Install the gem:

$ gem install tartan-grid

Install tartan SCSS:

$ tartan install --path app/assets/stylesheets

Or bundle the gem:

$ echo "gem 'tartan'" > Gemfile
$ bundle

Usage

Create a container element:

<div id='content'></div>
@import 'tartan/tartan';

#content {
  @include tartan-container;
}

Full-width content uses tartan-span-columns, partial-width content uses tartan-span-columns($column-count). By default, a container has 12 columns, so tartan-span-columns(6) is a 50%-width section.

<div id='content'>
  <header>
    <h1>Buy This Thing</h1>
  </header>

  <section id='pricing'>
    <div class='pricing-plan'>Pricing plan 1</div>
    <div class='pricing-plan'>Pricing plan 2</div>
  </section>
</div>
@import 'tartan/tartan';

#content {
  @include tartan-container;
}

header {
  @include tartan-span-columns;
}

.pricing-plan {
  @include tartan-span-columns(6);
  &:last-child { @include tartan-omega; }
}

For nested content, wrap content to be nested in tartan-sub-container($context). Further nested content should make use of the same $context: 

<div id='content'>
  <header>
    <h1>Buy This Thing</h1>
  </header>

  <section id='pricing'>
    <div class='pricing-plan'>Pricing plan 1</div>
    <div class='pricing-plan'>Pricing plan 2</div>
  </section>

  <aside>
    <h2>Hello from Sidebar</h2>
  </aside>

  <div id='primary-content'>
    <h2>Features</h2>

    <ul class='feature-list'>
      <li>Stuff you don't want</li>
      <li>Stuff you don't need</li>
    </ul>

    <ul class='feature-list'>
      <li>Stuff you don't want</li>
      <li>Stuff you don't need</li>
    </ul>
  </div>
</div>
@import 'tartan/tartan';

#content {
  @include tartan-container;
}

header {
  @include tartan-span-columns;
}

.pricing-plan {
  @include tartan-span-columns(6);
  &:last-child { @include tartan-omega; }
}

aside {
  @include tartan-span-columns(4);
}

#primary-content {
  @include tartan-sub-container(8);
  @include tartan-omega;

  h2 { @include tartan-span-columns(8, 8); }
}

.feature-list {
  @include tartan-span-columns(4, 8);
  &:last-child { @include tartan-omega; }
}

Options

After importing 'tartan/tartan' into your sass file, certain settings can be overridden:

$tartan-base-size (default: 16)
Inline-block rendering requires adjusting letter-spacing, so this should be set to the base font size of the page, in pixels.
$tartan-column-count (default: 12)
The default number of columns (i.e. the implicit $context) in the layout.
$tartan-column-width (default: 4em)
The default width of a single column.
$tartan-fluid (default: true)
Whether or not the layout should be fluid.
$tartan-gutter-width (default: 1em)
The default width of the gutter between columns.
$tartan-render-mode (default: inline-block)
When set to inline-block, uses inline-block layout rendering. Any other value uses floats.

Mixins

tartan-container
The wrapper for an entire grid, or set of $tartan-span-columns elements. When $tartan-render-mode is inline-block, its letter-spacing is a negative value, so all of its children must first be children of a $tartan-span-columns element.
tartan-sub-container($columns: $tartan-column-count, $context: $tartan-column-count)
A combination of tartan-container and tartan-span-columns, useful for creating complex nested layouts.
tartan-row
Useful when $tartan-render-mode is not inline-block, as it implements a clearfix.
tartan-span-columns($columns: $tartan-column-count, $context: $tartan-column-count, $tartan-render-mode: $tartan-render-mode)
The basic building block of tartan, states how many columns of a given context (12 by default) a container should use.
tartan-push-columns($columns: 1, $context: $tartan-column-count, $type: margin)
Pushes an element a number of columns to the right, with margin or padding. Can be overriden with tartan-no-push.
tartan-bumper($columns: 1, $context: $tartan-column-count, $type: margin)
Appends a number of columns onto the right of an element, with margin or padding
tartan-omega
Must be applied to the last element in a visual row, ensuring that it doesn't wrap. Can be overriden with tartan-no-omega (this is useful when implementing responsive layouts).

Functions

tartan makes available a number of functions that it uses internally to calculate layout.

tartan-columns-width($columns)
Returns in em units the width of a given number of columns in the current layout.
tartan-columns($columns, $context: $tartan-column-count)
Returns a percentage value representing the width of a number of columns in a given context.
tartan-gutter($context: $tartan-column-count)
Returns a percentage value representing the width of a gutter in a given context.
tartan-container-outer-width
Returns in em units the width of the entire container, based on $tartan-column-count.
tartan-relative-width($width, $context: $tartan-column-count)
Returns a percentage value representing how much of a layout $width (an em value) occupies in a given context.
